# Text to Any - 文本转通配类型

**节点功能：** `Text to Any` 将手写文本作为通配类型（`*`）输出，便于连接到更多输入端类型（例如部分节点的 COMBO 参数在连接时也可接收文本载荷）。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `text` | - | STRING | `` | 多行 | 输入文本 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `any` | * | 输出通配类型载荷，内容为输入文本 |

## 功能说明

- 通配输出：输出端使用通配类型 `*`，用于提升连接兼容性。
- 列表输入兼容：当 `text` 以列表形式输入时，节点取第一个元素作为输出。

## 典型用法

- 需要为下游节点提供文件名/标识符/提示词文本时，将 `any` 输出直接连接到对应输入端。

## 注意与建议

- 本节点仅负责传递文本内容，不对内容做格式校验；建议保持与下游节点期望的命名/路径/格式一致。
