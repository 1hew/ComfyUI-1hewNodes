# text_match_rownum - 多行文本行号匹配

**节点功能：** `text_match_rownum` 在多行文本中查找目标文本，并返回首个匹配行的行号（从 1 开始）。若未找到则返回 `0`。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `text_multiline` | - | STRING | - | - | 多行文本输入（按换行分割） |
| `text_single` | - | STRING | - | - | 要匹配的单行文本 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `rownum` | INT | 首个匹配行号（1-based）；未匹配返回 `0` |

## 功能说明

- 匹配逻辑按“去首尾空白后的整行相等”判断。
- 只返回第一个匹配结果，适合做分支判断或索引定位。
- 输入为空或异常时安全回退为 `0`。

## 典型用法

- 文本路由：将多行候选词与单个目标词匹配，输出行号给后续 `switch` 类节点。
- 配置选择：通过行号映射不同参数组或处理分支。

## 注意与建议

- 行号从 1 开始，不是 0 开始。
- 如果需要“包含匹配/正则匹配”，建议在前置节点先做文本预处理后再用本节点精确定位。
