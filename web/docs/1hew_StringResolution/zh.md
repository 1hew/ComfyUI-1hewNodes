# String Resolution - 分辨率标签选择器

**节点功能：** `String Resolution` 根据输入图像面积输出最接近的分辨率档位标签；如果未连接图像，则直接输出手动选择的标签。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `selection` | - | COMBO | `1k` | `0.5k` / `1k` / `2k` / `4k` | 手动选择的分辨率标签；未连接 `image` 时直接输出此值。 |
| `image` | 可选 | IMAGE | - | - | 输入图像批次，用于逐张推断最接近的分辨率档位。 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `string` | STRING | 分辨率标签字符串；批量输入时按批次顺序以换行拼接输出。 |

## 功能说明

- 面积匹配：按图像总像素面积在 `0.5k`、`1k`、`2k`、`4k` 档位中选择最近值。
- 手动回退：未连接图像时直接透传手动选择的标签。
- 支持批处理：图像批次中的每张图都会输出一行分辨率标签。
- 轻量辅助节点：适合给下游需要字符串分辨率参数的节点提供输入。

## 典型用法

- 在需要 `0.5k` / `1k` / `2k` / `4k` 字符串参数的模型或图像节点前，先把图像尺寸转换成分辨率标签。
- 不连接 `image` 时，可直接当作固定分辨率字符串选择器使用。

## 注意与建议

- 匹配依据是总像素面积，不考虑具体宽高比。
- 该节点只输出字符串标签，不会修改图像尺寸。
