# Multi Index Select - 多列统一索引选择

**节点功能：** `Multi Index Select` 支持动态 `input_1..input_N` 输入端口，使用同一个 `index` 对每一列分别取对应元素。可同时处理列表数据（如文本列表）和张量批次（如图像/遮罩批次）。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `index` | - | INT | 0 | -999999..999999 | 所有输入列共享的索引；支持负索引 |
| `input_1` | - | ANY (`*`) | - | - | 第一列输入 |
| `input_2…input_N` | 可选（动态，最多10个） | ANY (`*`) | - | - | 其他输入列，与 `output_2…output_N` 一一对应 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `output_1…output_N` | ANY (`*`) | 每列按同一 `index` 选出的结果；输入输出按后缀编号一一对应 |

## 功能说明

- 列映射稳定：`input_k` 始终对应 `output_k`。
- 统一索引：一个 `index` 同时控制所有列。
- 混合类型支持：
- 张量批次：按 batch 维取第 `index` 项，并保持 `B=1` 形状。
- 列表/元组：按下标取单个元素。
- 标量/其他类型：原样透传。
- 越界保护：索引会自动夹到有效范围；负索引按从尾部计数处理。

## 典型用法

- 图文同步取值：`image_list` 与 `string_list` 用同一索引同步选择。
- 混合输入形式：可在同一节点里同时接 list 与 batch。
- 多列并行选择：用一个节点替代多个单列索引节点。

## 注意与建议

- 本节点主要用于列表/批次数据；标量输入不会被切片。
- 若将字符串输出接到图像预览口会类型不匹配，请把不同类型输出接到对应节点。
