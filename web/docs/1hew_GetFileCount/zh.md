# Get File Count - 文件计数

**节点功能：** `Get File Count` 用于扫描指定目录并返回匹配的图片、视频、PSD 或文本文件数量，并额外输出列表形式的 `filename` 与 `file_path`。`filename` 可通过参数控制是否保留后缀。支持递归扫描、稳定排序，以及用于缓存的变化哈希。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `folder` | - | STRING | `""` | - | 目标目录路径。 |
| `type` | - | COMBO | `image` | `image` / `video` / `psd` / `txt` | 计数的文件类型组。 |
| `filename_suffix` | - | BOOLEAN | `false` | - | 控制 `filename` 是否保留扩展名；关闭时仅输出文件主名，开启后输出带后缀文件名。 |
| `include_subdir` | - | BOOLEAN | `true` | - | 扫描时包含子目录。 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `count` | INT | 匹配到的文件数量。 |
| `folder` | STRING | 输入目录回传。 |
| `filename` | STRING LIST | 匹配到的文件名列表，每个文件作为一个独立列表项输出。 |
| `file_path` | STRING LIST | 匹配到的路径列表，每个文件作为一个独立列表项输出。 |
| `include_subdir` | BOOLEAN | 输入选项回传。 |

## 功能说明

- 扩展名过滤：覆盖常用图片、视频、PSD 与文本格式扩展名。
- 稳定排序：路径按大小写无关排序，结果一致性更好。
- 文件名输出：按排序结果输出 `filename` 字符串列表，并可切换为“含后缀”或“仅名字”模式。
- 路径输出：按排序结果输出 `file_path` 字符串列表，便于下游稳定定位文件。
- 变化跟踪：以“路径列表哈希”作为缓存变化依据。

## 典型用法

- 在按序加载目录图片/视频前，先获取数量用于索引与循环控制。
- 在扫描 PSD 素材目录后，可将 `filename` 直接作为字符串列表传给支持 list 的下游节点，无需再经过 `List Custom String`。
- 当只需要不带后缀的素材名时，保持 `filename_suffix=false` 即可直接得到适合文本匹配的主文件名列表。
- 当需要把结果传给下游路径处理逻辑时，优先使用 `file_path`，避免递归扫描时丢失子目录信息。

## 注意与建议

- 当目录为空或未匹配到扩展名时，`count` 输出为 `0`。
- `filename` 返回的是文件名本身；当启用子目录扫描且不同子目录存在同名文件时，列表中会重复出现同名项。
- `filename_suffix=false` 时，仅移除最后一段扩展名，例如 `a.test.psd` 会输出为 `a.test`。
- `file_path` 返回扫描结果中的路径列表，可直接用于需要精确文件定位的下游节点或脚本。
