# Get File Count - 文件计数

**节点功能：** `Get File Count` 用于扫描指定目录并返回匹配的图片或视频文件数量。支持递归扫描、稳定排序，以及用于缓存的变化哈希。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `folder` | - | STRING | `""` | - | 目标目录路径。 |
| `type` | - | COMBO | `image` | `image` / `video` | 计数的文件类型组。 |
| `include_subdir` | - | BOOLEAN | `true` | - | 扫描时包含子目录。 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `count` | INT | 匹配到的文件数量。 |
| `folder` | STRING | 输入目录回传。 |
| `include_subdir` | BOOLEAN | 输入选项回传。 |

## 功能说明

- 扩展名过滤：覆盖常用图片与视频格式扩展名。
- 稳定排序：路径按大小写无关排序，结果一致性更好。
- 变化跟踪：以“路径列表哈希”作为缓存变化依据。

## 典型用法

- 在按序加载目录图片/视频前，先获取数量用于索引与循环控制。

## 注意与建议

- 当目录为空或未匹配到扩展名时，`count` 输出为 `0`。
