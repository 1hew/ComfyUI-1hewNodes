# Image to URL - 图像转 URL

**节点功能：** `Image to URL` 将 `IMAGE` 张量转换为可直接传递给下游节点或外部 API 的 URL 字符串。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `image` | - | IMAGE | - | - | 输入图像，支持单图或批量 |
| `mode` | 下拉框 | COMBO | `auto` | `auto` / `kefan` / `data` | 转换模式：自动回退、仅上传到 `kefan.cn`，或直接输出 data URL |
| `timeout` | - | INT | 30 | 5-300 | 上传请求超时时间（秒） |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `url` | STRING | URL 输出；单图时返回单个 URL，多图时按每行一个 URL 拼接为多行字符串 |

## 功能说明

- `auto`：优先上传到 `kefan.cn` 获取公网 URL；如果上传失败，则自动回退为 data URL。
- `kefan`：仅上传到 `kefan.cn`，成功时返回公网 URL，失败时报错。
- `data`：直接生成 `data:image/...;base64,...` 格式字符串，不依赖第三方上传服务。
- 批量输入支持：当输入是 image batch 时，会逐张转换，并按“一行一个 URL”的形式合并到同一个字符串输出。
- 带缓存复用：同一张图重复上传时会优先复用缓存映射，减少重复上传。

## 典型用法

- 将 ComfyUI 生成的图像转成 `data` 模式输出，传给支持图片 URL / data URL 的大模型或 HTTP 请求节点。
- 将图像切到 `kefan` 模式后，传给只接受公网图片地址的下游服务。
- 默认使用 `auto`，优先拿公网 URL，失败时也不中断后续流程。
- 批量输入多张图时，可直接把输出当作多行文本继续传给文本处理节点。

## 注意与建议

- `kefan` 依赖外网与第三方上传服务可用性；如果更看重稳定性，建议优先使用 `data`。
- 若下游节点只接受单个 URL，请确保输入为单图，或先对多行结果做拆分处理。
