# Text Match Value - 文本值匹配

**节点功能：** `Text Match Value` 在多行文本（键值对）中匹配指定的单行文本，并返回对应的值。支持精确匹配和前缀匹配，适用于简单的字典查找或条件路由。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `text_multiline` | - | STRING | - | - | 包含键值对的多行文本（格式如 `key: value` 或 `{key}: {value}`） |
| `text_single` | - | STRING | - | - | 需要匹配的单行查询文本 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `value` | * (ANY) | 匹配到的对应值，未匹配到则返回空字符串 |

## 功能说明

- 键值对解析：按行解析 `text_multiline`，支持冒号（`:` 或 `：`）分隔的键值对。
- 括号解包：自动去除键和值外层的大括号 `{}`，方便与通配符或特定格式配合使用。
- 匹配策略：
  1. 首先尝试精确匹配（忽略大小写和首尾空格）。
  2. 如果没有精确匹配，则尝试前缀匹配（即键是否以查询文本开头）。
- 容错处理：未匹配到任何内容时，安全地返回空字符串。

## 典型用法

- 提示词映射：根据简单的标签（如 "style_a"）输出完整的提示词段落。
- 参数路由：根据输入的字符串条件，选择对应的数值或配置字符串输出。

## 注意与建议

- 键和值之间的分隔符必须是英文冒号 `:` 或中文冒号 `：`。
- 匹配过程是大小写不敏感的。
