# Image Resize GPT Image 2.0 - GPT Image 2.0 尺寸适配

**节点功能：** `Image Resize GPT Image 2.0` 用于将输入图像/遮罩调整到 GPT Image 2.0 可接受的尺寸。`auto*` 与 Gemini 系列保持一致，会从固定预设表里选择最接近的尺寸；`dynamic*` 则按 GPT Image 2.0 规则尽量保持输入比例并动态缩放。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `preset_size` | - | COMBO | `auto (2k)` | `auto*` / `dynamic*` / 各预设尺寸项 | 目标尺寸选择；`auto*` 从预设表匹配，`dynamic*` 保持输入比例动态缩放，具体预设项输出固定尺寸 |
| `fit` | - | COMBO | `crop` | `crop` / `pad` / `stretch` | 适应模式：裁剪、填充、拉伸 |
| `pad_color` | - | STRING | `1.0` | 灰度/HEX/RGB/颜色名/`edge`/`average`/`extend`/`mirror` | `pad` 模式背景填充策略 |
| `image` | 可选 | IMAGE | - | - | 输入图像批次 |
| `mask` | 可选 | MASK | - | - | 输入遮罩批次，可与图像同步变换 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `image` | IMAGE | 调整后的图像批次 |
| `mask` | MASK | 与输出尺寸对齐的遮罩批次 |

## 功能说明

- 预设 auto：`auto` 会在全部预设中匹配最接近的尺寸；`auto (1k)` / `auto (2k)` / `auto (4k)` 会限制在对应档位的预设里匹配。
- 动态 dynamic：`dynamic` 会按输入面积选择最接近的 `1k` / `2k` / `4k` 档位并保持输入比例动态缩放；`dynamic (1k)` / `dynamic (2k)` / `dynamic (4k)` 会固定档位。
- 固定预设：提供 `1k`、`2k`、`4k` 三档下常用比例的稳定尺寸，适合需要明确输出尺寸的工作流。
- GPT Image 2.0 规则：动态尺寸与预设尺寸都会按 16 像素对齐、安全比例范围与最大边 3840 等规则处理。
- 支持无图像或无遮罩场景：只输入 `image` 或 `mask` 也可执行；都为空时输出目标尺寸背景图与全白遮罩。
- 三种适配模式：
  - `crop`：先按比例中心裁剪再缩放到目标尺寸。
  - `pad`：先按比例缩放再居中填充到目标尺寸。
  - `stretch`：直接拉伸到目标尺寸。

## 典型用法

- 送入 GPT Image 2.0 编辑前对齐到常见预设：`preset_size=auto (2k)`，`fit=crop`。
- 尽量保持输入原始比例：`preset_size=dynamic (2k)`，`fit=pad` 或 `fit=crop`。
- 需要固定横图尺寸：选择 `[2k] 2048x1152 (16:9)` 或 `[4k] 3840x2160 (16:9)`。
- 保留完整画面并补边：`fit=pad`，按需要设置 `pad_color`（如 `1.0`、`#000000`、`mirror`）。
- 只用遮罩生成对应画布：只连接 `mask` 输入，节点会生成对应背景图并输出对齐后的遮罩。

## 注意与建议

- 这张表是按 GPT Image 2.0 尺寸规则归纳出的节点固定预设，不是官方固定枚举表。
- `auto*` 会从这张表里挑选最近尺寸；`dynamic*` 会按输入比例动态计算。
- `4k` 尺寸生成可能耗时较长，下游 API 节点建议提高 `timeout_sec`。
