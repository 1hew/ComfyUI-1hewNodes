# Save Txt - 保存文本到磁盘

**节点功能：** `Save Txt` 将任意输入内容转换为文本并写入 `.txt` 文件，同时返回保存后的绝对路径。支持相对/绝对路径、自动编号、覆盖保存、`encode` 选择，以及仅预览不落盘的模式。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `any` | - | ANY (`*`) | - | - | 要保存的输入内容；会自动转换为字符串。字典/列表会格式化为 JSON 文本。 |
| `filename` | - | STRING | `txt/ComfyUI` | - | 保存文件前缀或绝对路径。相对路径会写入 ComfyUI 输出目录下。 |
| `auto_increment` | - | BOOLEAN | `true` | - | 为 `true` 时自动编号生成新文件；为 `false` 时使用固定文件名并覆盖。 |
| `save_output` | - | BOOLEAN | `true` | - | 为 `true` 时写入磁盘；为 `false` 时仅在 UI 中预览文本，不保存文件。 |
| `encode` | - | COMBO | `utf-8` | `utf-8` / `utf-8-sig` / `gbk` | 写入文本时使用的编码。 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `file_path` | STRING | 保存后的绝对路径；当 `save_output=false` 时返回空字符串。 |

## 功能说明

- 通用输入：支持任意类型输入，统一转换为文本；结构化数据会格式化为易读 JSON。
- 路径兼容：支持相对路径和绝对路径；相对路径默认保存到输出目录下。
- 自动编号：开启 `auto_increment` 时按 `_00001_` 形式递增，避免覆盖已有文件。
- 覆盖模式：关闭 `auto_increment` 时固定写入 `filename.txt`。
- 预览模式：`save_output=false` 时不写磁盘，但仍可在 UI 面板查看文本内容。

## 典型用法

- 将提示词、日志或 JSON 结果保存为文本文件，便于外部流程读取。
- 在调试阶段关闭 `save_output`，仅查看节点实际输出文本。

## 注意与建议

- 绝对路径模式下请确保目标目录具有写入权限。
- 批量并发保存时，节点内部会串行分配文件路径，减少重名冲突。
