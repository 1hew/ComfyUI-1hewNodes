# Image Batch Interleave - 图像批次分段交错重排

**节点功能：** `Image Batch Interleave` 将输入图像批次按 `segment_count` 拆成若干连续分段，再按“每段第 1 张、每段第 2 张……”的列优先顺序重新输出。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `image` | - | IMAGE | - | - | 输入图像批次 `(B, H, W, C)` |
| `segment_count` | - | INT | `2` | `1-100000` | 将输入批次分成多少段，再按列交错重排 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `image` | IMAGE | 重排后的图像批次 |

## 功能说明

- 连续分段：按原顺序把批次切成 `segment_count` 个连续小段。
- 列优先交错：先输出每段第 1 张，再输出每段第 2 张，以此类推。
- 尽量平均：当总帧数不能整除段数时，前面的分段会多 1 张，保证所有图片都被保留。
- 纯重排：仅调整批次索引顺序，不修改图像像素内容。

## 典型示例

- 输入 `1, 2, 3, 4, 5, 6`，`segment_count=3`
  - 分段后：`(1, 2)`、`(3, 4)`、`(5, 6)`
  - 输出后：`1, 3, 5, 2, 4, 6`

- 输入 `1, 2, 3, 4, 5, 6, 7`，`segment_count=3`
  - 分段后：`(1, 2, 3)`、`(4, 5)`、`(6, 7)`
  - 输出后：`1, 4, 6, 2, 5, 7, 3`

## 注意与建议

- 当 `segment_count=1` 时，输出顺序与输入一致。
- 当 `segment_count` 大于输入批次数量时，实际会按批次数量处理，每段最多 1 张图，因此输出也会保持原顺序。
- 该节点适合把连续批次打散为“跨段轮询”的顺序，便于后续拼接、预览或节奏重排。
