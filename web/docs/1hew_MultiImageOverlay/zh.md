# Multi Image Overlay - 多图层叠加

**节点功能：** `Multi Image Overlay` 按顺序依次叠加多个图层。RGBA 输入使用 alpha 通道进行正常的图层合成，RGB 输入视为不透明图层。任一输入带 alpha 时，节点会自动按 RGBA 处理并输出 RGBA。支持动态输入端口 `image_1..image_N` 与多种尺寸适应模式。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `fit_mode` | - | COMBO | `center` | `top_left` / `center` / `stretch` | 尺寸适应模式 |
| `color` | - | STRING | `1.0` | - | 兼容旧工作流保留的背景色参数；当任一输入带 alpha 时，节点会直接输出 RGBA |
| `image_1..image_N` | - | IMAGE | - | - | 动态输入的图像图层，`image_1` 为最上层，数字越大越靠底层 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `image` | IMAGE | 叠加合成后的图像批次 |

## 功能说明

- 动态端口：以 `image_` 开头的输入端口支持自动扩展，`image_1` 为最上层，编号越大越靠底层。
- 尺寸对齐：以最底层图像的尺寸为基准画布尺寸。
- 通道规则：任一输入带 alpha 时，所有图层都会按 RGBA 参与合成，输出自动保留 alpha。
- 适应模式：
  - `center`：居中放置，超出部分裁剪，不足部分透明。
  - `top_left`：左上角对齐放置。
  - `stretch`：拉伸至画布尺寸。
- Alpha 合成：正确处理 RGBA 图像的预乘 alpha 混合。RGB 图像视为完全不透明。

## 典型用法

- 图像合成：将带有透明背景的多个元素（如贴纸、水印、前景主体）叠加到背景图上。
- 批量叠加：支持批处理，当各图层批次大小不一致时，自动广播对齐到最大批次。

## 注意与建议

- 数字越大的输入越适合作为背景或底图；`image_1` 更适合作为最上层元素。
- 如果希望结果保持透明信息，只需保证任一输入本身带有 alpha。
