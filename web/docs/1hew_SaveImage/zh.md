# Save Image - 保存图像到磁盘

**节点功能：** `Save Image` 将 IMAGE 图像批次写入 ComfyUI 输出目录或临时目录，并返回保存后的绝对路径列表。支持文件名前缀配置，以及 prompt/workflow 元数据写入开关。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `image` | 可选 | IMAGE | - | - | 要保存的图像批次；输入为空时节点输出空结果。 |
| `filename_prefix` | - | STRING | `image/ComfyUI` | - | 保存文件前缀，交由 ComfyUI 路径生成处理；通常支持日期占位符（如 `%date:yyyy-MM-dd%`）。 |
| `save_output` | - | BOOLEAN | `true` | - | 开启时保存到输出目录；未开启时保存到临时目录。 |
| `save_metadata` | - | BOOLEAN | `true` | - | 开启时写入 prompt 与 workflow 相关元数据。 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `file_path` | STRING | 保存后的绝对路径，按换行分隔，每行对应一张图片。 |

## 功能说明

- 输出位置控制：通过 `save_output` 选择输出目录或临时目录。
- 元数据控制：通过 `save_metadata` 控制 `prompt` 与 `extra_pnginfo` 写入。
- 命名一致性：依托 ComfyUI 的路径分配逻辑生成唯一文件名。
- 界面预览：在 UI 面板展示保存结果的图片列表。

## 典型用法

- 最终结果保存到输出目录，中间过程保存到临时目录便于清理。
- 导出到外部流程时关闭元数据写入，获得更简洁的文件内容。

## 注意与建议

- 节点返回的是绝对路径；批量结果采用换行分隔。
- 节点对路径分配过程做串行化处理，计数器更稳定。
