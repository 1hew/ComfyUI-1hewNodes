# Multi Switch Select - 多输出选择路由

**节点功能：** `Multi Switch Select` 使用 1-based 的 `select` 在动态 `input_1..input_N` 中选中一路，并将该输入值输出到同编号的 `output_k`。未被选中的其他输出保持为 `None`。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `select` | - | INT | 1 | 1-10 | 选择要路由的输入编号；`1` 对应 `input_1` |
| `input_1` | - | ANY (`*`) | - | - | 第一路输入 |
| `input_2…input_10` | 可选（动态） | ANY (`*`) | - | - | 其他候选输入 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `output_1…output_10` | ANY (`*`) | 仅与 `select` 对应的 `output_k` 会输出选中的值，其余输出为 `None` |

## 功能说明

- 1-based 选择：`select=1` 选择 `input_1`，`select=2` 选择 `input_2`。
- 对应路由：`input_k` 只会输出到 `output_k`，编号关系固定。
- 惰性求值：仅请求被选中的输入端；若该输入未连接，则不继续等待并直接保持对应输出为 `None`。
- 越界保护：`select` 会自动限制到 `1..10` 范围内。

## 典型用法

- 多分支占位输出：在一个节点里保留多路 `output_k`，供下游按编号连接。
- 分支互斥控制：通过整数选择某一路输入并让其他路保持空值。
- 动态路由：结合其他逻辑节点生成 `select` 值，切换不同资源或参数。

## 注意与建议

- 下游请连接对应的 `output_k`，不要假设所有输出都会有值。
- 如果所选输入未连接，目标输出将为 `None`。
