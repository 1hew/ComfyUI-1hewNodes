# Multi Image List - 多图像列表

**节点功能：** `Multi Image List` 从动态 `image_X` 输入收集图像，并输出为 `image_list`。输入会按编号顺序处理；每个 `image_X` 可以接普通图像批次，也可以接图像列表，最终都会拆成单张图像后加入列表。

## 输入

| 参数名称 | 入端选择 | 数据类型 | 默认值 | 取值范围 | 描述 |
| -------- | -------- | -------- | ------ | -------- | ---- |
| `image_1` | 可选 | IMAGE / IMAGE_LIST | - | - | 第一个图像输入，可接批次或列表 |
| `image_2...image_N` | 可选 | IMAGE / IMAGE_LIST | - | - | 通过动态端口添加的更多图像输入，可接批次或列表 |

## 输出

| 输出名称 | 数据类型 | 描述 |
|---------|----------|------|
| `image_list` | IMAGE_LIST | 按输入编号顺序输出的图像列表 |

## 功能说明

- 动态输入：可继续添加 `image_2`、`image_3` 等端口。
- 顺序稳定：按数字后缀从小到大收集输入。
- 批次/列表拆分：每个输入若为批次或列表，会递归拆成单张 `[1,H,W,C]` 后加入列表。
- 空输入跳过：未连接、空张量或非图像张量会被忽略。

## 典型用法

- 将多个独立图像端口转换为 `image_list`，方便接需要图像列表的后续节点。
- 和 `Image List to Batch` 搭配，在需要时再统一成批次。
